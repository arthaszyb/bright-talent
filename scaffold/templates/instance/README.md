# DE Instance Repository

This repository contains the configuration and knowledge base for a **Digital Employee** instance.

## Quick start

1. **Validate configuration:**
   ```bash
   ./de validate
   ```

2. **Build the runtime:**
   ```bash
   make build
   ```

3. **Start a conversation with the DE:**
   ```bash
   ./de start
   ```

## Key files

- `instance.yaml` — Identity, scope, settings, escalation, KB refs, bridge config.
- `skills.yaml` — Skill dependencies (resolved and pinned at build time).
- `.env` — Local secrets (gitignored; for production, use the Instance Manager).
- `kb/team/` — Team-specific runbooks, SOPs, incident postmortems.
- `runtime/` — Build output (gitignored; regenerated by `./de build`).

## Workflow

1. Edit `instance.yaml` to configure identity, scope, settings, escalation contacts, and bridge.
2. Edit `skills.yaml` to declare skill dependencies.
3. Add team knowledge under `kb/team/`.
4. Run `make build` to validate and render the runtime.
5. Run `./de start` to test locally.
6. Commit `instance.yaml`, `skills.yaml`, and `kb/team/` changes.
7. Push to trigger CI (see `.github/workflows/instance-ci.yml`).

## Maintenance

- **Update skills:** edit `skills.yaml` and rebuild.
- **Add runbooks:** create `.md` files under `kb/team/runbooks/` and update `kb/team/_index.md`.
- **Upgrade scaffold base:** `./de upgrade` checks for new versions.
- **Detect drift:** `./de diff` compares runtime against build manifest.

For more, see the project docs under `/docs/10-scaffold/` and `/docs/20-instance/`.
